# Шпаргалка по Git

Этот файл содержит основные и наиболее часто используемые команды Git, разбитые по категориям.

## 1. Первоначальная настройка (Конфигурация)
Эти команды обычно выполняются один раз после установки Git.

```bash
# Установить имя пользователя (будет отображаться в коммитах)
git config --global user.name "Твое Имя"

# Установить email пользователя
git config --global user.email "твой@email.com"

# Посмотреть все текущие настройки
git config --list
```

## 2. Создание и получение репозиториев
Начало работы с новым или существующим проектом.

```bash
# Инициализировать новый локальный репозиторий в текущей папке
git init

# Склонировать существующий репозиторий из интернета (например, с GitHub)
git clone <url_репозитория>
```

## 3. Повседневная работа (Изменения и коммиты)
Команды для сохранения твоей работы.

```bash
# Посмотреть состояние репозитория (какие файлы изменены, добавлены или удалены)
git status

# Добавить конкретный файл в индекс (подготовить к коммиту)
git add <имя_файла>

# Добавить ВСЕ измененные и новые файлы в индекс
git add .

# Создать коммит (сохранить изменения) с сообщением
git commit -m "Твое сообщение о том, что было сделано"

# Посмотреть историю коммитов (кто, когда и что сделал)
git log
# Более компактный вид истории:
git log --oneline
```

## 4. Ветвление (Branches)
Ветки позволяют работать над новыми фичами безопасно, не ломая основной код.

```bash
# Посмотреть список всех локальных веток
git branch

# Создать новую ветку
git branch <имя_ветки>

# Переключиться на другую ветку
git checkout <имя_ветки>
# (В новых версиях Git также можно использовать: git switch <имя_ветки>)

# Создать новую ветку и сразу переключиться на нее
git checkout -b <имя_ветки>
# (Или: git switch -c <имя_ветки>)

# Слить изменения из другой ветки в текущую (например, слить фичу в main)
git merge <имя_ветки>

# Удалить ветку (после того как она была слита)
git branch -d <имя_ветки>
```

## 5. Работа с удаленными репозиториями (Remote)
Обмен кодом с сервером (GitHub, GitLab и т.д.).

```bash
# Посмотреть список подключенных удаленных репозиториев
git remote -v

# Отправить твои локальные коммиты на сервер
git push origin <имя_ветки>

# Получить последние изменения с сервера и сразу слить их с локальной веткой
git pull origin <имя_ветки>

# Просто скачать изменения с сервера, но пока не сливать их с твоим кодом
git fetch
```

## 6. Отмена изменений (Если что-то пошло не так)
Как исправить ошибки.

```bash
# Отменить изменения в файле до состояния последнего коммита (ВНИМАНИЕ: изменения потеряются!)
git checkout -- <имя_файла>
# (Или в новых версиях: git restore <имя_файла>)

# Убрать файл из индекса (отменить git add), но сохранить изменения в самом файле
git reset HEAD <имя_файла>
# (Или: git restore --staged <имя_файла>)

# Отменить последний коммит, создав новый коммит, который делает всё наоборот
git revert <хэш_коммита>

# ПОЛНОСТЬЮ откатиться к старому коммиту, удалив все последующие (опасно!)
git reset --hard <хэш_коммита>
```

## 7. Разное полезное
```bash
# Посмотреть разницу между текущими измененными файлами и последним коммитом
git diff

# Посмотреть разницу между файлами в индексе (подготовленными) и последним коммитом
git diff --staged

# Временно спрятать незакомиченные изменения (чтобы, например, срочно переключиться на другую ветку)
git stash

# Вернуть спрятанные изменения обратно
git stash pop
```
